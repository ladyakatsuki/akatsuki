# CRT 显像管解密开场动画 Spec

## Why
需要一个具有强烈视觉冲击力与悬疑感的网页开场动画：以老式显像管（CRT）显示屏的雪花屏起手，过渡到满屏跳动的乱码，再以《黑客帝国》式的从左至右解密效果，逐步显现文字"你是谁"，最后弹出一个浏览器新窗口继续解密剧情。整体营造复古、神秘、电影感的交互体验。

## What Changes
- 新增单文件 HTML 网页 `index.html`，包含完整的 CRT 开场动画。
- 实现四阶段动画序列：雪花屏 → 满屏乱码 → 从左至右解密 → 弹出新窗口。
- 实现老式 CRT 视觉效果：屏幕弧度、扫描线、暗角、闪烁、磷光发光、显示器外壳边框。
- 实现乱码字符集：中文乱码使用锟/斤/拷/烫等 GBK 解码错误字符；英文乱码使用拉丁字母与西里尔字母混合。
- 实现从左至右的解密算法：解密光标扫过位置后字符锁定为正确字符，未扫过位置持续跳动乱码。
- 使用 Web Audio API 合成音效（雪花白噪声、打字咔哒声、电流嗡鸣），无外部音频资源。
- 使用 `window.open` 弹出真实浏览器新窗口，新窗口内继续播放解密动画显示后续剧情文字。
- 提供"点击启动"入口以满足浏览器自动播放策略与弹窗拦截策略（需用户手势）。

## Impact
- Affected specs: 无（全新项目）。
- Affected code: 新增 `index.html`（主页面，含全部 CSS/JS）；弹窗内容通过 Blob URL 内联生成，无需额外文件。
- 运行环境：现代浏览器（Chrome/Edge/Firefox），需支持 Canvas、Web Audio API、`window.open`。

## ADDED Requirements

### Requirement: 点击启动入口
系统 SHALL 在页面加载后显示一个"点击启动"提示覆盖层，等待用户点击后再开始动画与音频，以符合浏览器自动播放策略与弹窗拦截策略。

#### Scenario: 用户点击启动
- **WHEN** 用户点击启动覆盖层
- **THEN** 覆盖层消失，初始化 AudioContext，开始播放雪花屏阶段动画与白噪声音效

### Requirement: CRT 显像管视觉效果
系统 SHALL 将整个动画渲染在一个仿老式显像管显示屏的容器内，具备以下视觉特征：
- 屏幕弧度（轻微透视/圆角变形）
- 水平扫描线（repeating-linear-gradient 叠加）
- 四周暗角（radial-gradient 暗化）
- 整体轻微闪烁（opacity 微动画）
- 文字磷光发光（text-shadow / canvas shadowBlur，绿色荧光）
- 显示器外壳边框（深色塑料质感边框包裹屏幕）

#### Scenario: 视觉呈现
- **WHEN** 动画运行
- **THEN** 画面始终带有扫描线、暗角、闪烁与绿色磷光发光效果，并被 CRT 外壳边框包裹

### Requirement: 阶段一 雪花屏
系统 SHALL 在动画开始时显示满屏雪花噪点（TV static），并随时间逐渐淡出消失。

#### Scenario: 雪花淡出
- **WHEN** 雪花屏阶段开始
- **THEN** Canvas 持续绘制随机黑白噪点，雪花不透明度随时间从 1.0 线性衰减至 0，同时播放白噪声音效并随淡出减弱音量

### Requirement: 阶段二 满屏乱码
系统 SHALL 在雪花淡出过程中及淡出后，以满屏跳动的乱码字符填充屏幕，字符高速随机变化。

#### Scenario: 乱码跳动
- **WHEN** 进入满屏乱码阶段
- **THEN** 屏幕以等宽字符网格填充，每个字符位置以高频随机切换为乱码字符（中文乱码或英文乱码），整体呈现剧烈跳动效果

### Requirement: 乱码字符集
系统 SHALL 使用两组乱码字符集：
- 中文乱码：`锟斤拷烫锘锝锳锎锏锠锢锤锥锦锧锨锩锪锫锬锭键锰锱锲锳锴锶锷锸锹锺锻锼锽锾锿`（GBK 解码错误特征字符）
- 英文乱码：拉丁字母（A-Z、a-z）与西里尔字母（А-Яа-я）混合

#### Scenario: 字符选取
- **WHEN** 需要生成乱码字符
- **THEN** 从上述字符集中随机选取字符进行显示

### Requirement: 阶段三 从左至右解密
系统 SHALL 以"解密光标"从屏幕左侧扫至右侧，光标扫过的字符位置锁定为目标文字"你是谁"的对应字符，未扫过的位置继续跳动乱码。

#### Scenario: 解密推进
- **WHEN** 解密阶段运行
- **THEN** 光标位置以恒定速度从左向右推进；已扫过位置的字符稳定显示为目标字符并保持磷光；未扫过位置的字符继续随机跳动乱码
- **WHEN** 光标扫过全部目标字符
- **THEN** "你是谁"完整稳定显示，短暂停顿后进入弹窗阶段

### Requirement: 阶段四 弹出新窗口继续解密
系统 SHALL 在主屏显示"你是谁"后，使用 `window.open` 弹出一个真实浏览器新窗口，新窗口内继续播放解密动画，显示后续剧情文字。

#### Scenario: 弹窗成功
- **WHEN** "你是谁"显示完成并短暂停顿
- **THEN** 调用 `window.open` 打开新窗口，新窗口加载内联的解密动画页面，从乱码解密出后续剧情文字
- **WHEN** 弹窗被浏览器拦截
- **THEN** 在主屏显示提示"新窗口被拦截，请允许弹窗后重试"，并提供重试按钮

### Requirement: Web Audio 合成音效
系统 SHALL 使用 Web Audio API 实时合成以下音效，不依赖任何外部音频文件：
- 雪花白噪声（ScriptProcessorNode / AudioBufferSourceNode 填充随机值）
- 解密打字咔哒声（短促方波/噪声脉冲，随光标推进触发）
- 电流嗡鸣（低频正弦波持续背景音）

#### Scenario: 音效播放
- **WHEN** 各阶段运行
- **THEN** 播放对应音效；雪花淡出时白噪声音量同步衰减；解密推进时按节奏触发咔哒声

### Requirement: 绿色荧光配色
系统 SHALL 采用经典《黑客帝国》绿色荧光配色：纯黑底色（#000），文字与发光为荧光绿（约 #00ff66 / #33ff66），磷光发光效果。

#### Scenario: 配色一致
- **WHEN** 全程动画
- **THEN** 背景为纯黑，所有文字与发光元素为荧光绿，无其他主色干扰
